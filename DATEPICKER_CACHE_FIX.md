# راهنمای رفع مشکل Cache تقویم شمسی

## مشکل
اگر تغییرات تقویم شمسی اعمال نمی‌شود، احتمالاً مرورگر شما فایل‌های قدیمی را از cache بارگذاری می‌کند.

## راه‌حل

### 1. پاک کردن Cache مرورگر (توصیه می‌شود)

#### Chrome / Edge:
- `Ctrl + Shift + Delete` را فشار دهید
- "Cached images and files" را انتخاب کنید
- روی "Clear data" کلیک کنید

#### Firefox:
- `Ctrl + Shift + Delete` را فشار دهید
- "Cache" را انتخاب کنید
- روی "Clear Now" کلیک کنید

### 2. Hard Refresh (سریع‌تر)

- **Windows/Linux**: `Ctrl + F5` یا `Ctrl + Shift + R`
- **Mac**: `Cmd + Shift + R`

### 3. باز کردن در حالت Incognito/Private

- `Ctrl + Shift + N` (Chrome/Edge)
- `Ctrl + Shift + P` (Firefox)

## تأیید اینکه مشکل حل شده

بعد از پاک کردن cache:

1. صفحه ایجاد حراجی را باز کنید
2. روی فیلد "زمان شروع" کلیک کنید
3. سعی کنید تاریخ دیروز را انتخاب کنید
4. **نباید بتوانید** - روزهای گذشته باید خاکستری و غیرفعال باشند

## نکته برای توسعه‌دهندگان

فایل‌های CSS و JS حالا با `?v={{ time() }}` بارگذاری می‌شوند که cache را خودکار دور می‌زند.

## تست

برای تست سریع، این صفحه را باز کنید:
```
http://localhost/haraj/public/test-mindate.html
```

این صفحه شامل 3 تست است:
1. فیلد با minDate (نباید بتوان تاریخ گذشته انتخاب کرد)
2. فیلد با تاریخ پیش‌فرض (باید در باز شدن، تاریخ انتخاب شده نمایش داده شود)
3. فیلد عادی (بدون محدودیت)
