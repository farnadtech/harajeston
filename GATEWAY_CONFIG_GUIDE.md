# راهنمای تنظیم درگاه‌های پرداخت

## 1. زرین‌پال (ZarinPal)

### ثبت‌نام و دریافت Merchant ID
1. به سایت [zarinpal.com](https://www.zarinpal.com) بروید
2. ثبت‌نام کنید و وارد پنل شوید
3. از منوی "درگاه پرداخت" گزینه "مشخصات درگاه" را انتخاب کنید
4. Merchant ID را کپی کنید

### تنظیمات در پنل ادمین
- **نام درگاه**: ZarinPal
- **Merchant ID**: کد دریافتی از زرین‌پال (36 کاراکتر با فرمت UUID)
- **حالت تست**: برای محیط تست فعال کنید

### Sandbox (تست)
- Merchant ID تست: `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`
- در حالت sandbox نیازی به تایید حساب نیست
- پرداخت‌ها واقعی نیستند

---

## 2. زیبال (Zibal)

### ثبت‌نام و دریافت Merchant
1. به سایت [zibal.ir](https://www.zibal.ir) بروید
2. ثبت‌نام کنید و وارد پنل شوید
3. از منوی "تنظیمات" گزینه "درگاه پرداخت" را انتخاب کنید
4. Merchant Code را کپی کنید

### تنظیمات در پنل ادمین
- **نام درگاه**: Zibal
- **Merchant ID**: کد دریافتی از زیبال

### Sandbox (تست)
- Merchant تست: `zibal`
- این merchant برای تست رایگان است
- پرداخت‌ها واقعی نیستند

---

## 3. وندار (Vandar)

### ثبت‌نام و دریافت API Key
1. به سایت [vandar.io](https://vandar.io) بروید
2. ثبت‌نام کنید و وارد پنل شوید
3. از منوی "تنظیمات" گزینه "API" را انتخاب کنید
4. API Key را کپی کنید

### تنظیمات در پنل ادمین
- **نام درگاه**: Vandar
- **API Key**: کلید دریافتی از وندار

### نکات
- وندار نیاز به تایید حساب دارد
- برای تست باید با پشتیبانی تماس بگیرید

---

## 4. پی‌پینگ (PayPing)

### ثبت‌نام و دریافت Token
1. به سایت [payping.ir](https://www.payping.ir) بروید
2. ثبت‌نام کنید و وارد پنل شوید
3. از منوی "توکن‌ها" گزینه "ایجاد توکن جدید" را انتخاب کنید
4. Token را کپی کنید

### تنظیمات در پنل ادمین
- **نام درگاه**: PayPing
- **API Key**: توکن دریافتی از پی‌پینگ

### نکات
- پی‌پینگ نیاز به تایید حساب دارد
- برای تست باید حساب تایید شده داشته باشید

---

## مراحل فعال‌سازی در پنل ادمین

1. وارد پنل ادمین شوید
2. به مسیر `admin/payment-gateways` بروید
3. روی دکمه "ویرایش" درگاه مورد نظر کلیک کنید
4. اطلاعات درگاه را وارد کنید:
   - **Merchant ID** یا **API Key** (بسته به نوع درگاه)
   - **وضعیت**: فعال
5. روی "ذخیره تغییرات" کلیک کنید

## تست درگاه

### روش 1: اسکریپت تست
```
http://localhost/haraj/public/test-payment-flow.php
```

### روش 2: تست واقعی
1. به صفحه کیف پول بروید: `/wallet`
2. مبلغ دلخواه را وارد کنید (حداقل 10,000 تومان)
3. درگاه را انتخاب کنید
4. روی "افزایش موجودی" کلیک کنید
5. به درگاه پرداخت منتقل می‌شوید
6. در حالت تست، پرداخت را تایید کنید
7. به سایت برمی‌گردید و موجودی شارژ می‌شود

## خطاهای رایج

### "merchant یافت نشد" یا "API Key نامعتبر"
- مطمئن شوید اطلاعات درگاه را صحیح وارد کرده‌اید
- فضای خالی اضافی نداشته باشد
- از کپی مستقیم از پنل درگاه استفاده کنید

### "درگاه غیرفعال است"
- در پنل ادمین، وضعیت درگاه را "فعال" کنید
- مطمئن شوید حساب شما در درگاه تایید شده است

### "مبلغ نامعتبر"
- مبلغ باید بین 10,000 تا 100,000,000 تومان باشد
- مبلغ باید عدد صحیح باشد (بدون ممیز یا نقطه)

### "خطا در اتصال به درگاه"
- اتصال اینترنت را بررسی کنید
- مطمئن شوید سرور درگاه در دسترس است
- لاگ‌ها را بررسی کنید: `storage/logs/laravel.log`

---

## محاسبه مبلغ نهایی

مبلغ نهایی = مبلغ اصلی + مالیات (9%)

**مثال:**
- مبلغ درخواستی: 100,000 تومان
- مالیات (9%): 9,000 تومان
- مبلغ نهایی: 109,000 تومان

---

## امنیت

### نکات مهم:
1. هرگز اطلاعات درگاه را در کد قرار ندهید
2. از متغیرهای محیطی (`.env`) استفاده نکنید - از دیتابیس استفاده می‌شود
3. SSL/HTTPS را فعال کنید
4. Callback URL باید HTTPS باشد
5. IP سرور را در پنل درگاه ثبت کنید (در صورت نیاز)
