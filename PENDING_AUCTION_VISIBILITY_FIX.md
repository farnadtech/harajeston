# تنظیم نمایش حراجی‌های pending - فقط کنترل ادمین

## مشکل قبلی
- فروشندگان می‌توانستند با چک‌باکس `show_before_start` کنترل کنند که آیا حراجی قبل از شروع نمایش داده شود یا نه
- حتی اگر تنظیمات ادمین غیرفعال بود، اگر فروشنده چک‌باکس را فعال کرده بود، حراجی نمایش داده می‌شد
- این باعث عدم کنترل یکپارچه توسط ادمین می‌شد

## راه‌حل پیاده‌سازی شده

### 1. حذف کنترل فروشنده
- چک‌باکس `show_before_start` از فرم ایجاد حراجی ادمین حذف شد
- فیلد `show_before_start` دیگر در validation و ذخیره‌سازی استفاده نمی‌شود
- فیلد در دیتابیس باقی می‌ماند ولی نادیده گرفته می‌شود

### 2. کنترل کامل توسط ادمین
فقط تنظیمات ادمین (`default_show_before_start`) تعیین می‌کند که آیا حراجی‌های pending نمایش داده شوند:

#### اگر تنظیمات ادمین فعال باشد (✅):
- **همه** حراجی‌های pending برای **همه کاربران** نمایش داده می‌شوند
- کارت محصول با برچسب "X روز تا شروع" نمایش داده می‌شود
- دکمه "هنوز شروع نشده" غیرفعال است

#### اگر تنظیمات ادمین غیرفعال باشد (❌):
- حراجی‌های pending **فقط** برای ادمین و صاحب آگهی قابل مشاهده هستند
- کاربران عادی با دسترسی به لینک مستقیم، خطای 404 دریافت می‌کنند
- در لیست‌ها و جستجوها اصلاً نمایش داده نمی‌شوند

## فایل‌های تغییر یافته

### 1. `app/Http/Controllers/ListingController.php`
```php
// بررسی دسترسی برای آگهی‌های pending
if ($listing->status === 'pending') {
    // بررسی تنظیمات ادمین - فقط تنظیمات ادمین مهم است
    $showPendingListings = \App\Models\SiteSetting::get('default_show_before_start', false);
    
    if (!$showPendingListings) {
        // فقط ادمین و صاحب آگهی می‌تونن ببینن
        if (!auth()->check() || 
            (auth()->user()->role !== 'admin' && auth()->id() !== $listing->seller_id)) {
            abort(404);
        }
    }
}
```

### 2. `app/Http/Controllers/Admin/ListingController.php`
- حذف `'show_before_start' => 'nullable|boolean'` از validation
- حذف `'show_before_start' => $validated['show_before_start'] ?? $defaultShowBeforeStart` از create array

### 3. `resources/views/admin/listings/create.blade.php`
- حذف چک‌باکس "نمایش حراجی قبل از شروع در لیست‌ها و صفحات سایت"
- فقط چک‌باکس "تمدید خودکار" باقی ماند

## ویژگی‌های موجود

### 1. برچسب زمان تا شروع
کارت محصول برای حراجی‌های pending نمایش می‌دهد:
- "X روز تا شروع" (اگر بیشتر از 1 روز مانده باشد)
- "X ساعت تا شروع" (اگر کمتر از 1 روز مانده باشد)
- "X دقیقه تا شروع" (اگر کمتر از 1 ساعت مانده باشد)
- "در حال شروع..." (اگر کمتر از 1 دقیقه مانده باشد)

### 2. مرتب‌سازی "زودتر شروع می‌شود"
در صفحات جستجو و دسته‌بندی، گزینه مرتب‌سازی "زودتر شروع می‌شود" وجود دارد که:
- حراجی‌های pending را بر اساس زمان شروع مرتب می‌کند
- حراجی‌هایی که زودتر شروع می‌شوند اول نمایش داده می‌شوند

## تست

### فایل تست
`public/test-pending-visibility.php`

این فایل نمایش می‌دهد:
- وضعیت تنظیمات ادمین
- لیست حراجی‌های pending
- آیا هر حراجی نمایش داده می‌شود یا نه (بر اساس تنظیمات ادمین)

### مراحل تست

1. **تست با تنظیمات فعال:**
   - به پنل ادمین بروید: `/admin/settings`
   - تیک "نمایش پیش‌فرض حراجی‌ها قبل از شروع" را بزنید
   - ذخیره کنید
   - به صفحه اصلی بروید: `/`
   - باید حراجی‌های pending با برچسب "X روز تا شروع" نمایش داده شوند

2. **تست با تنظیمات غیرفعال:**
   - به پنل ادمین بروید: `/admin/settings`
   - تیک "نمایش پیش‌فرض حراجی‌ها قبل از شروع" را بردارید
   - ذخیره کنید
   - از حساب کاربری خارج شوید (logout)
   - به صفحه اصلی بروید: `/`
   - حراجی‌های pending نباید نمایش داده شوند
   - اگر لینک مستقیم یک حراجی pending را باز کنید، باید خطای 404 دریافت کنید

3. **تست دسترسی ادمین:**
   - با حساب ادمین وارد شوید
   - حتی اگر تنظیمات غیرفعال باشد، باید بتوانید حراجی‌های pending را ببینید

4. **تست دسترسی فروشنده:**
   - با حساب فروشنده وارد شوید
   - حتی اگر تنظیمات غیرفعال باشد، باید بتوانید حراجی‌های خودتان را ببینید

## نکات مهم

1. ⚠️ فیلد `show_before_start` در دیتابیس باقی می‌ماند ولی دیگر استفاده نمی‌شود
2. 🎯 فقط تنظیمات ادمین کنترل می‌کند که آیا pending ها نمایش داده شوند
3. ✅ فروشندگان هیچ کنترلی روی نمایش حراجی‌های pending ندارند
4. 🔒 ادمین و صاحب آگهی همیشه می‌توانند حراجی‌های pending را ببینند
5. 📊 در لیست‌ها و جستجوها، حراجی‌های pending فقط اگر تنظیمات ادمین فعال باشد نمایش داده می‌شوند
