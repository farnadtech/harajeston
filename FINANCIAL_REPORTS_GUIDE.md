# راهنمای سیستم گزارشات مالی

## خلاصه
سیستم گزارشات مالی امکان مشاهده و تحلیل کامل درآمد سایت، کمیسیون‌ها، و آمار معاملات را فراهم می‌کند.

## دسترسی به گزارشات

### برای ادمین
1. وارد پنل مدیریت شوید (`/admin/dashboard`)
2. از منوی سمت راست، روی "گزارشات مالی" کلیک کنید
3. صفحه گزارشات مالی باز می‌شود

## بخش‌های گزارشات

### 1. موجودی کیف پول سایت
نمایش موجودی فعلی کیف پول سایت:
- **موجودی کل**: کل مبلغ موجود در کیف پول
- **مبلغ فریز شده**: مبلغی که قفل شده (مثلاً سپرده‌ها)
- **قابل برداشت**: مبلغی که می‌توان برداشت کرد

### 2. خلاصه درآمد
آمار کلی درآمد در بازه زمانی انتخاب شده:
- **کل درآمد**: مجموع کمیسیون‌ها و سپرده‌های ضبط شده
- **کمیسیون‌ها**: کل کمیسیون‌های دریافتی از معاملات
- **سپرده‌های ضبط شده**: سپرده‌هایی که از برندگان عدم پرداخت ضبط شده
- **معاملات موفق**: تعداد معاملات تکمیل شده

### 3. نمودار درآمد روزانه
نمودار خطی نمایش درآمد روزانه با سه خط:
- خط آبی: کمیسیون‌ها
- خط قرمز: سپرده‌های ضبط شده
- خط سبز: کل درآمد

### 4. آمار کلی پلتفرم
- کل کاربران
- تعداد فروشندگان
- تعداد خریداران
- حراج‌های فعال

### 5. فروشندگان برتر
لیست 5 فروشنده برتر بر اساس:
- کل درآمد
- تعداد فروش

### 6. خریداران برتر
لیست 5 خریدار برتر بر اساس:
- کل مبلغ خرید
- تعداد خرید

### 7. آمار دسته‌بندی‌ها
جدول نمایش آمار هر دسته‌بندی:
- تعداد حراج
- ارزش کل
- میانگین قیمت

## فیلتر بازه زمانی

می‌توانید بازه زمانی دلخواه را انتخاب کنید:
1. تاریخ شروع را وارد کنید
2. تاریخ پایان را وارد کنید
3. روی "اعمال فیلتر" کلیک کنید

**پیش‌فرض**: ماه جاری

## دانلود گزارش (CSV)

برای دانلود گزارش به صورت فایل CSV:
1. بازه زمانی دلخواه را انتخاب کنید
2. روی دکمه "دانلود CSV" کلیک کنید
3. فایل CSV شامل درآمد روزانه دانلود می‌شود

## جزئیات کمیسیون‌ها

برای مشاهده جزئیات تمام کمیسیون‌ها:
1. از صفحه اصلی گزارشات، روی "جزئیات کمیسیون‌ها" کلیک کنید
2. جدول کامل تراکنش‌های کمیسیون نمایش داده می‌شود

اطلاعات نمایش داده شده:
- تاریخ و ساعت
- شرح تراکنش
- مبلغ کمیسیون
- موجودی قبل و بعد

## محاسبات

### کل درآمد
```
کل درآمد = کمیسیون‌ها + سپرده‌های ضبط شده
```

### نرخ کمیسیون
```
نرخ کمیسیون = (کل کمیسیون‌ها / حجم کل معاملات) × 100
```

### میانگین کمیسیون
```
میانگین کمیسیون = کل کمیسیون‌ها / تعداد معاملات موفق
```

## منابع درآمد

### 1. کمیسیون معاملات
- کمیسیون از خریدار
- کمیسیون از فروشنده
- کمیسیون مشترک (از هر دو)

### 2. سپرده‌های ضبط شده
- سپرده برندگانی که در مهلت مقرر پرداخت نکرده‌اند
- به عنوان جریمه به کیف پول سایت واریز می‌شود

## نکات مهم

### کیف پول سایت
- کمیسیون‌ها به کیف پول کاربر با `user_id = 1` واریز می‌شود
- این کاربر باید از قبل ایجاد شده باشد
- اگر این کاربر وجود نداشته باشد، گزارشات خالی نمایش داده می‌شود

### بازه زمانی
- تمام محاسبات بر اساس `created_at` تراکنش‌ها انجام می‌شود
- می‌توانید هر بازه زمانی دلخواه را انتخاب کنید
- برای مقایسه ماه‌ها، از فیلتر استفاده کنید

### عملکرد
- گزارشات با استفاده از query های بهینه شده محاسبه می‌شوند
- برای بازه‌های زمانی بزرگ، ممکن است چند ثانیه طول بکشد
- نمودارها با Chart.js رندر می‌شوند

## API Endpoints

### دریافت گزارش اصلی
```
GET /admin/financial-reports
Parameters: start_date, end_date
```

### دریافت جزئیات کمیسیون‌ها
```
GET /admin/financial-reports/commissions
Parameters: start_date, end_date
```

### دانلود CSV
```
GET /admin/financial-reports/export
Parameters: start_date, end_date
```

### دریافت داده‌های نمودار (AJAX)
```
GET /admin/financial-reports/chart-data
Parameters: type (daily/monthly), start_date, end_date
```

## مثال‌های کاربردی

### مثال 1: گزارش ماه جاری
```
بازه زمانی: 1403/12/01 تا 1403/12/30
کل درآمد: 5,000,000 تومان
کمیسیون‌ها: 4,500,000 تومان
سپرده‌های ضبط شده: 500,000 تومان
معاملات موفق: 90
میانگین کمیسیون: 50,000 تومان
نرخ کمیسیون: 5%
```

### مثال 2: مقایسه دو ماه
```
ماه قبل:
- کل درآمد: 3,000,000 تومان
- معاملات: 60

ماه جاری:
- کل درآمد: 5,000,000 تومان
- معاملات: 90

رشد: +66.7% درآمد، +50% معاملات
```

## عیب‌یابی

### گزارشات خالی است
**علت**: کیف پول سایت (user_id = 1) وجود ندارد
**راه‌حل**: 
```php
$siteUser = User::find(1);
if (!$siteUser) {
    $siteUser = User::create([
        'id' => 1,
        'name' => 'سایت',
        'email' => 'site@example.com',
        'password' => bcrypt('password'),
        'role' => 'admin',
    ]);
    $siteUser->wallet()->create(['balance' => 0, 'frozen' => 0]);
}
```

### نمودار نمایش داده نمی‌شود
**علت**: Chart.js لود نشده است
**راه‌حل**: اطمینان حاصل کنید که CDN Chart.js در دسترس است

### اعداد فارسی نمایش داده نمی‌شود
**علت**: helper `@persian()` کار نمی‌کند
**راه‌حل**: اطمینان حاصل کنید که `PersianNumberService` به درستی تنظیم شده است
