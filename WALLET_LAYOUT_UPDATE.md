# بهروزرسانی Layout صفحه کیف پول

## تغییرات انجام شده

صفحه کیف پول (`resources/views/wallet/show.blade.php`) به گونه‌ای بهروزرسانی شد که بر اساس نقش کاربر، از layout مناسب استفاده کند.

### قبل از تغییرات
- همه کاربران (خریدار، فروشنده، ادمین) از `layouts.app` استفاده می‌کردند
- صفحه برای همه یکسان بود

### بعد از تغییرات

#### 1. انتخاب Layout بر اساس نقش
```php
@if(auth()->user()->role === 'admin')
    @extends('layouts.admin')
@elseif(auth()->user()->canSell())
    @extends('layouts.seller')
@else
    @extends('layouts.app')
@endif
```

#### 2. عنوان صفحه متناسب با نقش
- **Admin**: "مدیریت کیف پول" - "مشاهده و مدیریت موجودی کیف پول"
- **Seller**: "کیف پول من" - "مدیریت موجودی و تراکنش‌های مالی"
- **Buyer**: عنوان داخل محتوا نمایش داده می‌شود

#### 3. Container مناسب
- **Buyer**: با container و padding کامل (`max-w-7xl mx-auto py-8 px-4`)
- **Seller & Admin**: بدون container اضافی (چون layout خودشان sidebar دارند)

## نتیجه

### برای خریدار (Buyer)
- از `layouts.app` استفاده می‌کند
- هدر اصلی سایت با منوی کامل
- محتوا در container با padding مناسب

### برای فروشنده (Seller)
- از `layouts.seller` استفاده می‌کند
- Sidebar فروشنده در سمت راست
- هدر با عنوان "کیف پول من"
- محتوا بدون container اضافی

### برای ادمین (Admin)
- از `layouts.admin` استفاده می‌کند
- Sidebar ادمین در سمت راست
- هدر با عنوان "مدیریت کیف پول"
- محتوا بدون container اضافی

## فایل‌های تغییر یافته
- `resources/views/wallet/show.blade.php`

## تست
برای تست تغییرات:
1. به عنوان خریدار وارد شوید و `/wallet` را باز کنید
2. به عنوان فروشنده وارد شوید و `/wallet` را باز کنید
3. به عنوان ادمین وارد شوید و `/wallet` را باز کنید

یا از فایل تست استفاده کنید:
```
http://localhost/haraj/public/test-wallet-layout.php
```

## نکات مهم
- تمام قابلیت‌های صفحه (افزایش موجودی، برداشت، تاریخچه) برای همه نقش‌ها یکسان است
- فقط layout و نمایش تغییر کرده است
- کد منطقی و عملکرد صفحه تغییری نکرده است
