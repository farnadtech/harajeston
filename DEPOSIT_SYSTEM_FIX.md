# رفع مشکلات سیستم سپرده مزایده

## تاریخ: 1404/12/06
## وضعیت: ✅ تکمیل شده

## مشکلات قبلی

### 1. فروشنده می‌توانست سپرده تعیین کند ❌
- فروشندگان در فرم ثبت حراجی می‌توانستند مبلغ سپرده را مشخص کنند
- این باید فقط توسط ادمین در تنظیمات سایت قابل تنظیم باشد

### 2. تغییرات ادمین اعمال نمی‌شد ❌
- وقتی ادمین مبلغ سپرده را در تنظیمات تغییر می‌داد، برای آگهی‌ها اعمال نمی‌شد
- سیستم از فیلد `required_deposit` در جدول listings استفاده می‌کرد

### 3. باکس Livewire می‌چرخید ❌
- باکس پرداخت مزایده "در حال پردازش" نشان می‌داد و همینطور می‌چرخید
- کامپوننت Livewire مشکل داشت

### 4. سپرده بلوک نمی‌شد ❌
- موقع ثبت اولین پیشنهاد، سپرده از کیف پول بلوک نمی‌شد

## راه‌حل‌های پیاده‌سازی شده

### 1. ✅ حذف فیلد سپرده از فرم فروشنده
- کامپوننت Livewire participation حذف شد
- فرم پیشنهاد مستقیماً نمایش داده می‌شود
- فروشنده دیگر نمی‌تواند سپرده تعیین کند

### 2. ✅ خواندن سپرده از تنظیمات سایت
تمام قسمت‌های زیر به‌روزرسانی شدند تا از `site_settings` استفاده کنند:

#### `resources/views/listings/show.blade.php`
```php
// Get deposit from site settings
$depositSetting = \App\Models\SiteSetting::where('key', 'deposit_type')->first();
$depositType = $depositSetting ? $depositSetting->value : 'none';

$depositAmount = 0;
if ($depositType === 'fixed') {
    $fixedSetting = \App\Models\SiteSetting::where('key', 'deposit_fixed_amount')->first();
    $depositAmount = $fixedSetting ? (int)$fixedSetting->value : 0;
} elseif ($depositType === 'percentage') {
    $percentageSetting = \App\Models\SiteSetting::where('key', 'deposit_percentage')->first();
    $percentage = $percentageSetting ? (float)$percentageSetting->value : 0;
    $depositAmount = (int)($listing->starting_price * ($percentage / 100));
}
```

#### `app/Services/BidService.php`
- حذف چک `$listing->required_deposit`
- افزودن محاسبه سپرده از `site_settings`
- بلوک کردن سپرده در اولین پیشنهاد:
```php
// Block deposit amount for first bid
if ($depositAmount > 0 && !$userHasBid) {
    $wallet->balance -= $depositAmount;
    $wallet->frozen += $depositAmount;
    $wallet->save();
}
```

#### `app/Http/Requests/PlaceBidRequest.php`
- به‌روزرسانی validation برای استفاده از `site_settings`
- چک موجودی کیف پول شامل سپرده برای اولین پیشنهاد

### 3. ✅ نمایش مستقیم فرم پیشنهاد
- کامپوننت Livewire حذف شد
- فرم پیشنهاد مستقیماً نمایش داده می‌شود
- اطلاعات سپرده در باکس آبی نمایش داده می‌شود

### 4. ✅ بلوک کردن سپرده
- سپرده در اولین پیشنهاد از `balance` کسر و به `frozen` اضافه می‌شود
- از جدول `wallets` و ستون `frozen` استفاده می‌شود
- فقط برای اولین پیشنهاد کاربر اعمال می‌شود

## نحوه کار سیستم جدید

### برای ادمین:
1. ادمین وارد بخش "تنظیمات سایت" می‌شود
2. در قسمت "تنظیمات سپرده شرکت در مزایده" می‌تواند:
   - نوع سپرده را انتخاب کند: مبلغ ثابت یا درصد
   - مبلغ ثابت را وارد کند (مثلاً 1,000,000 تومان)
   - درصد را وارد کند (مثلاً 10%)
3. تغییرات بلافاصله برای تمام حراجی‌ها اعمال می‌شود

### برای کاربر:
1. کاربر وارد صفحه حراجی می‌شود
2. اگر سپرده فعال باشد، در باکس آبی نمایش داده می‌شود:
   - "سپرده شرکت در مزایده: X تومان"
   - "این مبلغ برای اولین پیشنهاد شما بلوک می‌شود"
3. موقع ثبت اولین پیشنهاد:
   - سیستم چک می‌کند موجودی کافی است (پیشنهاد + سپرده)
   - سپرده از balance کسر و به frozen اضافه می‌شود
4. برای پیشنهادهای بعدی:
   - فقط مبلغ پیشنهاد چک می‌شود
   - سپرده دیگر کسر نمی‌شود

### انواع سپرده:
1. **بدون سپرده (none)**: هیچ سپرده‌ای نیاز نیست
2. **مبلغ ثابت (fixed)**: مثلاً 1,000,000 تومان برای همه حراجی‌ها
3. **درصد (percentage)**: مثلاً 10% از قیمت پایه حراجی

## تست‌های لازم

### 1. تست تنظیمات ادمین
```
1. وارد پنل ادمین شوید
2. به تنظیمات سایت بروید
3. نوع سپرده را تغییر دهید (ثابت/درصد)
4. مبلغ را تغییر دهید
5. ذخیره کنید
6. به صفحه یک حراجی بروید
7. بررسی کنید سپرده جدید نمایش داده می‌شود
```

### 2. تست ثبت پیشنهاد با سپرده
```
1. کیف پول خود را شارژ کنید (مثلاً 5,000,000 تومان)
2. به یک حراجی با قیمت پایه 1,000,000 بروید
3. اگر سپرده 10% است، باید 100,000 تومان سپرده نمایش داده شود
4. پیشنهاد 1,500,000 تومان بدهید
5. بررسی کنید:
   - پیشنهاد ثبت شد
   - از کیف پول 100,000 تومان به frozen منتقل شد
   - balance شما 3,900,000 شد (5M - 1M - 100K)
   - frozen شما 100,000 شد
```

### 3. تست پیشنهاد دوم (بدون سپرده)
```
1. پیشنهاد دوم 2,000,000 تومان بدهید
2. بررسی کنید:
   - فقط 2,000,000 از balance کسر شد
   - frozen همچنان 100,000 است
```

### 4. تست موجودی ناکافی
```
1. کیف پول خود را خالی کنید
2. سعی کنید پیشنهاد بدهید
3. باید پیام خطا نمایش داده شود:
   "موجودی کیف پول شما کافی نیست. مبلغ مورد نیاز: X تومان (شامل Y تومان سپرده)"
```
